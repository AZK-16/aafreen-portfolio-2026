Streamlining East Midlands Railway’s Ticketing Experience

The Problem
EMR’s system had usability challenges when searching for destinations and paying for tickets.
The Outcome
Did a conceptual redesign of EMR’s interface that improved navigation and accessibility.
Domain: Public Transport
Time to Results: 3 months
Role: UX Designer & Researcher
2 Context

EMR ticket machines are used by thousands of commuters daily to buy tickets for public transport services like National Rail.

3 Challenge

EMR’s system therefore needs to be:

Fast
Commuters need to buy tickets quickly and catch their train on time.
Easy to Understand
Using the system should not cause cognitive overload, especially in the train station’s already busy and fast-paced environment.
Accessible
The system should accommodate both local and international commuters, regardless of their familiarity with the ticketing process.

5 Key Insights
Insight 1
Usability challenges when searching destinations
Evidence
The screen used slow arrow-based navigation for browsing results and limited recognition of alternative station names
Recommendations
Scrollbar replaces arrow-based navigation
Search by alternative station names and codes e.g. ‘EUS’ for Euston
Insight 2
Confusing labels on ‘Payment’ screen
Evidence
Some commuters were confused when paying by card - one commuter tapped their card in the wrong place, and another gave up and paid by cash instead
Recommendation
Update the on-screen machine image and labels so it naturally maps to the real EMR machine
6 Design Challenge 1: Search Destinations
6.1 Exploring Wireframes

I began by exploring established layouts and features in well-known transport services like EMR, such as National Rail. This enabled me to identify usability issues in the current layout and their potential solutions.

Current Layout Usability Issues
Cognitive Overload
Large ‘One Step Back’, ‘Cancel’ and ‘Delete’ buttons not needed
Slow up and down arrow-based navigation
Proposed Layouts
Option 1
Search Results Next to Search Bar
Option 2
Search Results Under Search Bar
6.2 Final Choice: Why Option 1: Results Next to Search Bar?
Familiarity
Although TfL’s desktop site displays search results beneath the search bar, the original EMR interface presents them alongside it. Retaining this layout in the redesigned interface helps preserve familiarity for existing users.
Easier to Check Results
Keeping the keyboard under the search bar makes it easier to check keyboard inputs and the corresponding update in the search bar.

7 Design Challenge 2: Payment Labels & Mirrored Layout
Current Layout Issues
Image Doesn’t Match Actual Machine
Confusing to know where to pay
Repeated payment option labels are overwhelming
Proposed Layouts
Left-to-Right Layout
Intuitive Labels & Represent Real Machine
Clearly labelled each payment option so it’s easy to find
Image resembles real machine including EMR screen, so commuters know payment options’ positions relative to screen
Right-to-Left Layout
Balance Between Reading Direction & Consistency
Mirrored layout so that RTL language speakers read the ‘Please pay’ message first
Kept EMR banner in English for consistency
Kept English numerals to match currency used in region
8 Project Reflection
Constraint
Limited Observation Time & Location

Due to time constraints, we observed a single station over one morning and two evening rush hours to capture various user interactions.

Improvements
What We’d Do Differently Next Time
Observe more EMR stations
Observe quieter periods to see how users behave differently under less time pressure
Longer observation period e.g. 1 week
